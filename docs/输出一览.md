# pty-agent 输出一览

> 列出所有可能的输出场景，包含 JSON 模式和自然语言模式的具体输出格式。

---

## 1. 输出模式

| 模式 | 启用方式 | stdout 内容 | stderr 内容 |
|------|----------|-------------|-------------|
| **JSON 模式**（默认） | 默认 / 不加 `--output-by-natural-language` | **仅 JSON**，每行一个 JSON 对象 | 无（守护进程日志不泄漏） |
| **自然语言模式** | `--output-by-natural-language` | 程序输出 + 元数据（纯文本） | `--color` 启用时元数据走 stderr |

---

## 2. 命令响应输出

### 2.1 exec / send / read — result 类型

守护进程返回 `type` 为 `exec` / `send` / `read` 的 result 响应。

#### JSON 模式

```json
{"type":"exec","session_id":"myid","output":"Python 3.11.9 ...\n>>> ","output_offset":1234,"trigger_matched":true,"reason":"matched","program":{"command":"python -i -u","start_time":"2026-06-22T14:32:15.47","running":true,"pty_type":"subprocess"},"debug":{"processes":[{"pid":1234,"path":"C:\\Python311\\python.exe"}],"gui_windows":[],"pending_events":[{"time":"2026-06-22T14:32:15.47","type":"process_spawn","pid":1234,"info":"PID 1234 created"}]}}
```

`--no-debug` 时移除 `debug` 字段：

```json
{"type":"exec","session_id":"myid","output":"Python 3.11.9 ...\n>>> ","output_offset":1234,"trigger_matched":true,"reason":"matched","program":{"command":"python -i -u","running":true,"pty_type":"subprocess"}}
```

**reason 取值**：

| reason | 含义 |
|--------|------|
| `matched` | 触发正则匹配命中 |
| `timeout` | 等待超时 |
| `idle_timeout` | 输出静默超时 |
| `ended` | 程序退出 |
| `crashed` | 进程崩溃 |
| `gui_detected` | 检测到 GUI 窗口 |
| `ok` | 无触发条件，直接返回（read 命令） |

**program 字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `command` | str/list | 启动命令 |
| `running` | bool | 程序是否仍在运行 |
| `pty_type` | str | PTY 后端类型：`subprocess` / `win-conpty` / `unix-pty` / `none` |
| `start_time` | str | ISO 8601 启动时间（可选） |
| `exit_code` | int | 退出码（仅程序退出时存在） |
| `error_message` | str | 错误信息（仅异常时存在） |

**debug 字段**（`--no-debug` 时移除）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `processes` | list | 进程树 `[{"pid":int,"path":str}]` |
| `gui_windows` | list | GUI 窗口 `[{"hwnd":int,"pid":int,"title":str,"class_name":str}]` |
| `pending_events` | list | 待处理事件 |

**warning 字段**（可选）：会话已结束等警告信息。

#### 自然语言模式

```
Python 3.11.9 ...
>>>

# ── session ────────────────────────
# matched | running
# session id: myid
# pty type: subprocess
# program: python -i -u
# started at: 2026-06-22 14:32:15
# current time: 2026-06-22 14:32:20

# ── debug ────────────────────────
# process tree: PID 1234 (C:\Python311\python.exe)

# ── events (1) ────────────
# [+]  [14:32:15] PID 1234 created

# ── offset ────────────────────────
# 1234
```

各段说明：

| 段 | 条件 | 内容 |
|----|------|------|
| 程序输出 | `output` 非空 | 终端实际输出文本（stdout） |
| session 段 | 有触发/状态信息 | `matched`/`timeout`/`ended` 等状态 + 会话元数据 |
| ⚠ 警告 | `warning` 非空 | 警告信息（如"会话已结束"） |
| debug 段 | 有进程树/GUI 窗口 | 进程树列表 + GUI 窗口详情 |
| events 段 | 有待处理事件 | 事件列表（`[+]` 创建 / `[-]` 退出 / `[!!]` 崩溃 / `[W]` GUI） |
| error 段 | `error_message` 非空 | 错误信息 |
| offset 段 | `output_offset > 0` | 输出偏移量 |

---

### 2.2 list — ok 类型（会话列表）

#### JSON 模式

有会话：
```json
{"type":"ok","sessions":[{"id":"web","command":"python -i -u","running":true,"pending_events":3},{"id":"gcc","command":"g++ main.cpp","running":false,"pending_events":0}]}
```

无会话：
```json
{"type":"ok","sessions":[],"note":"无活跃会话"}
```

#### 自然语言模式

有会话：
```
# ── sessions (2) ──────
#   [web] python -i -u  [running, pending 3]
#   [gcc] g++ main.cpp  [ended]
```

无会话：
```
# no active sessions
```

---

### 2.3 events — ok 类型（事件查询）

#### JSON 模式

```json
{"type":"ok","session_id":"myid","pending_events":[{"time":"2026-06-22T14:32:15.47","type":"process_spawn","pid":11888,"info":"PID 11888 created","still_active":false},{"time":"2026-06-22T14:32:16.89","type":"process_crash","pid":14708,"info":"PID 14708 crashed! exit=1","still_active":false}],"count":2}
```

会话已结束时附带 `warning`：
```json
{"type":"ok","session_id":"myid","pending_events":[...],"count":2,"warning":"会话已结束（旧会话数据）"}
```

#### 自然语言模式

```
# ── events (2) ────────────
# [+]  [14:32:15] PID 11888 created
# [!!] [14:32:16] process crashed!
#    PID 14708 crashed! exit=1
```

崩溃事件使用 `════════════` 分隔：
```
# ════════════ process crashes ════════════
# [!!] [14:32:16] process crashed!
#    PID 14708 crashed! exit=1
```

---

### 2.4 kill — ok 类型

#### JSON 模式

```json
{"type":"ok"}
```

#### 自然语言模式

无输出。

### 2.5 closewin — ok 类型

#### JSON 模式

成功：
```json
{"type":"ok","closed":true,"hwnd":1313450}
```

失败：
```json
{"type":"ok","closed":false,"hwnd":1313450}
```

#### 自然语言模式

成功：
```
window 0x001403AA closed
```

失败：
```
window 0x001403AA close failed
```

---

### 2.6 stop — ok 类型

#### JSON 模式

```json
{"type":"ok"}
```

#### 自然语言模式

无输出（守护进程信息由 `stop_daemon()` 的 `_safe_print` 输出，见第 3 节）。

### 2.7 error 类型

所有错误响应格式统一。

#### JSON 模式

```json
{"type":"error","error":"会话 'xxx' 不存在"}
```

常见错误消息：

| 场景 | error 内容 |
|------|-----------|
| 会话不存在 | `会话 'xxx' 不存在` |
| 会话已结束 | `会话 'xxx' 已结束，请先 kill 后重新 exec` |
| 缺少会话 id | `缺少会话 id` |
| 缺少 command | `缺少 command 参数` |
| 认证失败 | `认证失败` |
| 未知指令 | `未知指令类型: xxx` |
| JSON 解析失败 | `请求格式错误: JSON 解析失败` |
| 服务器内部错误 | `服务器内部错误` |
| 写入输入失败 | `写入输入失败` |
| 启动会话失败 | `启动会话失败` |
| 参数过长 | `参数 'xxx' 过长（最多 N 字符）` |
| 无效正则 | `无效的正则表达式: xxx` |
| 无效行范围 | `无效的行范围: xxx` |
| 缺少 hwnd | `缺少 hwnd 参数` |
| 关闭窗口失败 | `关闭窗口失败` |

#### 自然语言模式

```
error: 会话 'xxx' 不存在
```

输出到 stderr。

## 3. 守护进程生命周期信息（info 类型）

由 `start_daemon()` / `stop_daemon()` 内部 `_safe_print()` 输出。在 `exec`/`send`/`read`/`list` 等命令自动启动守护进程时也会触发。

### JSON 模式

所有信息以 `{"type":"info","message":"..."}` 格式输出到 stdout：

| 场景 | message |
|------|---------|
| 守护进程已在运行 | `[pty-agent] 守护进程已在运行中 (端口 12345)` |
| 守护进程已启动 | `[pty-agent] 守护进程已启动 (端口 12345)` |
| 可用 shell 信息 | `[pty-agent] 可用 shell: cmd (...), powershell (...), pwsh (...), bash (不可用)` |
| 守护进程启动失败（超时） | `[pty-agent] 守护进程启动失败（超时），端口 12345 可能已被占用` |
| 守护进程未运行 | `[pty-agent] 守护进程未运行` |
| 停止守护进程失败 | `[pty-agent] 停止守护进程失败` |
| TCP 停止失败 | `[pty-agent] TCP 停止失败: <异常>` |
| 已强制终止 | `[pty-agent] 已强制终止守护进程 (PID 12345)` |
| 强制终止失败 | `[pty-agent] 强制终止失败: <异常>` |
| 守护进程已停止 | `[pty-agent] 守护进程已停止` |
| 守护进程启动失败（异常） | `[pty-agent] 守护进程启动失败: <异常>` |

示例（`exec` 自动启动守护进程，stdout 逐行输出）：
```json
{"type":"info","message":"[pty-agent] 守护进程已启动 (端口 54482)"}
{"type":"info","message":"[pty-agent] 可用 shell: cmd (...), powershell (...)"}
{"type":"exec","session_id":"py","output":"Python 3.11.9 ...\n>>> ","output_offset":0,"trigger_matched":true,"reason":"matched","program":{...}}
```

### 自然语言模式

纯文本输出到 stdout：

```
[pty-agent] 守护进程已启动 (端口 54482)
[pty-agent] 可用 shell: cmd (...), powershell (...), pwsh (...)
```

## 4. 非命令响应输出

以下输出不来自守护进程响应，由客户端 `__main__.py` 直接生成。

### 4.1 配置查询 — config 类型（`--show-config`）

#### JSON 模式

```json
{"type":"config","content":"当前调用配置:\n  debug = on\n  encoding = (未设置)\n  keep_ansi = off\n  newline = off\n  output_by_natural_language = off\n  timeout = 120.0\n  # 注: 这些默认值仅在有子命令（如 exec/send）时生效，仅作查询参考"}
```

#### 自然语言模式

```
当前调用配置:
  debug = on
  encoding = (未设置)
  keep_ansi = off
  newline = off
  output_by_natural_language = off
  timeout = 120.0
  # 注: 这些默认值仅在有子命令（如 exec/send）时生效，仅作查询参考
```

---

### 4.2 帮助文本 — help 类型（无子命令）

#### JSON 模式

```json
{"type":"help","content":"usage: pty-agent [-h] [--show-config [KEY]] ...\n\n命令行交互式程序交互代理\n\npositional arguments:\n  {start,stop,list,exec,send,read,kill,events,closewin}\n                        可用命令\n..."}
```

#### 自然语言模式

argparse 默认帮助文本输出到 stdout。

---

### 4.3 警告信息 — warning 类型

#### `--default` 单独使用

JSON 模式：
```json
{"type":"warning","message":"警告: --default 仅在与子命令（如 exec/send）配合时有效，单独使用不会产生效果\n  已设置临时值: timeout = 30"}
```

自然语言模式：输出到 stderr。

#### `--idle-after-first-output` 缺少 `--idle-timeout`

JSON 模式：
```json
{"type":"warning","message":"--idle-after-first-output 需要配合 --idle-timeout 使用，单独设置无效（当前未启用静默超时检测）"}
```

自然语言模式：输出到 stderr。

#### `--default` 键名无效

JSON 模式：
```json
{"type":"error","error":"无效的配置键: 'xxx'"}
```

自然语言模式：输出到 stderr，然后 `sys.exit(1)`。

---

### 4.4 客户端异常

#### KeyboardInterrupt（Ctrl+C）

JSON 模式：
```json
{"type":"error","error":"操作被用户中断"}
```

自然语言模式：`操作被用户中断` 输出到 stderr。

#### 未捕获异常

JSON 模式：
```json
{"type":"error","error":"<异常信息>"}
```

自然语言模式：`错误: <异常信息>` 输出到 stderr。

---

### 4.5 客户端连接错误（transport.py）

以下错误输出到 **stderr**（不影响 JSON 模式的 stdout 纯度）：

| 场景 | stderr 输出 |
|------|------------|
| 启动守护进程失败 | `error: failed to start daemon` |
| 重启守护进程失败 | `error: failed to restart daemon` |
| 连接失败 | `error: <异常信息>` |

---

### 4.6 守护进程无响应

当守护进程 TCP 连接成功但无响应时：

JSON 模式：
```json
{"type":"error","error":"daemon not responding"}
```

自然语言模式：`error: daemon not responding` 输出到 stderr。

## 5. 完整输出流程示例

### 5.1 首次 exec（自动启动守护进程）

**JSON 模式** stdout 输出（3 行 JSON）：
```json
{"type":"info","message":"[pty-agent] 守护进程已启动 (端口 54482)"}
{"type":"info","message":"[pty-agent] 可用 shell: cmd (...), powershell (...), pwsh (...)"}

```

**自然语言模式** stdout 输出：
```
[pty-agent] 守护进程已启动 (端口 54482)
[pty-agent] 可用 shell: cmd (...), powershell (...), pwsh (...)

>>>
```

**JSON 模式** stdout 输出：

**JSON 模式** stdout 输出：

**JSON 模式** stdout 输出：

### 5.5 kill 命令

**JSON 模式** stdout 输出：
```json
{"type":"ok"}
```

### 5.6 stop 命令

**JSON 模式** stdout 输出：
```json
{"type":"ok"}
{"type":"info","message":"[pty-agent] 守护进程已停止"}
```

## 6. JSON 模式输出类型汇总

| type | 来源 | 说明 |
|------|------|------|
| `exec` / `send` / `read` | 守护进程 result 响应 | 命令执行结果 |
| `ok` | 守护进程 ok 响应 | kill / stop / list / events / closewin 成功 |
| `error` | 守护进程 / 客户端 | 错误响应 |
| `info` | `lifecycle._safe_print()` | 守护进程启停信息 |
| `config` | `__main__.py` | `--show-config` 配置查询 |
| `help` | `__main__.py` | 无子命令时的帮助文本 |
| `warning` | `__main__.py` | 参数警告 |
| `pong` | 守护进程 | ping 响应（内部使用，不输出到 CLI） |
