## 运行方式

### pytest 测试（单元 + 集成）

```powershell
# 全部测试
python -m pytest test/ -v

# 仅单元测试
python -m pytest test/unit/ -v

# 仅集成测试
python -m pytest test/integration/ -v

# 指定文件
python -m pytest test/unit/test_job.py -v
```

### Protocol 双模式测试

可直接运行（独立于 pytest）：

```powershell
python test/unit/protocol/test_ansi.py
python test/unit/protocol/test_encoding.py
python test/unit/protocol/test_message.py
```

### GDB 端到端测试

需要系统 PATH 中可用的 gdb.exe（GDB 17+）+ `test_debug.exe`：

```powershell
cd test/test_gdb_full
python test_gdb_full.py
```

## 测试要求

### 单元测试

- **隔离**：不依赖外部进程、网络、文件系统
- **快速**：单测试 < 1s
- **平台标记**：Windows 专用测试用 `@pytest.mark.skipif` 跳过非 Win32
- **fixture**：使用 `@pytest.fixture` 管理 setUp/tearDown
- **断言**：使用 `assert`，非 `self.assert*`

### 集成测试

- **真实子进程**：创建实际的 Python 子进程验证退出码
- **超时保护**：所有等待循环带 deadline，超时后 `pytest.fail`
- **清理**：`autouse` fixture 确保测试结束后清理子进程
- **时长**：单测试 < 5s

### GDB 端到端测试

- **完整流程**：守护进程生命周期 → GDB 会话 → 断点/单步/变量 → 边界场景
- **独立环境**：自启动/停止守护进程，不依赖外部状态
- **不包含**：不测试 GUI 窗口检测（测试环境无显示器）

## 添加新测试

1. 判断测试类型：
   - 单一类/函数、可 mock → `test/unit/`
   - 多模块协作、需子进程 → `test/integration/`
   - 完整 daemon 交互 → 脚本形式（参考 `test_gdb_full.py`）
2. 按被测模块的文件名命名：`test_<模块名>.py`
3. pytest 测试函数以 `test_` 开头
4. 添加 Doxygen 风格的模块文档字符串
