# pty-agent 命令行交互参考

> 完整命令参考，面向**终端用户**。开发指南见 `doc/instruction.md`。

---

## 通用选项

以下选项适用于所有子命令（`start` / `stop` / `exec` / `send` / `read` / `list` / `events` / `kill` / `closewin`），**放子命令前后均可**：

| 选项 | 说明 |
|------|------|
| `--color` | 启用终端着色输出（默认关闭，元数据走 stderr） |
| `--output-by-natural-language` | 启用自然语言输出（默认 JSON 格式） |
| `--encoding <编码>` | 指定终端编码，本次调用记忆 |
| `--no-debug` | 禁用响应中的 debug 输出（进程树/GUI 窗口/事件） |
| `--show-config [KEY]` | 查看当前调用配置值 |
| `--default <KEY> <VALUE>` | 临时覆盖本次调用的默认配置（可用键：`output-by-natural-language`/`timeout`/`newline`/`keep-ansi`/`encoding`/`debug`，`<VALUE>` 是配置值或 `on`/`off`；不持久化） |

示例：
```powershell
#  放子命令前面
pty-agent --output-by-natural-language send py "1+1" -t ">>>"
pty-agent --encoding utf-8 --color exec test -c "cmd" -t "ok"

#  禁用 debug 输出
pty-agent --no-debug exec test -c "cmd" -t "ok"
pty-agent exec test -c "cmd" -t "ok" --no-debug

#  放子命令后面
pty-agent send py "1+1" -t ">>>" --output-by-natural-language
pty-agent exec test -c "cmd" -t "ok" --color

#  混着放
pty-agent --encoding utf-8 exec test -c "cmd" -t "ok" --output-by-natural-language
```

## 临时配置管理

### 设置临时配置

```powershell
pty-agent --default timeout 30                           # 临时超时 30 秒
pty-agent --default newline on                            # 临时仅换行后检查触发
pty-agent --default keep-ansi on                          # 临时保留 ANSI 转义序列
pty-agent --default output-by-natural-language on          # 临时自然语言输出
pty-agent --default encoding gbk                           # 临时终端编码 gbk
pty-agent --default debug off                              # 临时禁用 debug 输出
```

> **注意**：`--default` 仅在与子命令（如 `exec`/`send`）配合时有效。单独使用会打印警告并无效。所有 `--default` 设置仅影响本次调用，不持久化。

```powershell
pty-agent exec build -c "gcc main.cpp" --default timeout 60   # 
pty-agent --default timeout 60 exec build -c "gcc main.cpp"   #  放前面也行
pty-agent --default timeout 30                                  #  警告：单独使用无效
```

### 查看配置

```powershell
pty-agent --show-config                    # 查看全部配置
pty-agent --show-config timeout             # 查看单项配置
pty-agent --show-config encoding
```

展示的配置值仅作查询参考，实际上只有配合子命令时才会生效。配置为纯内存，不持久化。

## 命令一览

```
pty-agent [通用选项] start                        启动守护进程
pty-agent [通用选项] stop                         停止守护进程
pty-agent [通用选项] list                         列出所有会话
pty-agent [通用选项] exec <id> -c "<cmd>"         启动会话
pty-agent [通用选项] send <id> "<输入>"            发送输入
pty-agent [通用选项] read <id>                    读取输出
pty-agent [通用选项] events <id>                  查看所有事件
pty-agent [通用选项] kill <id>                    终止会话
pty-agent [通用选项] closewin <id> <hwnd>         关闭 GUI 窗口
```

> 自动启动：首次执行 `exec`/`send`/`read`/`list`/`events` 等命令时，若守护进程未运行，客户端自动启动守护进程。不再需要手动 `start`。

## 命令详解

### start / stop

```powershell
pty-agent start        # 启动后台守护进程
pty-agent stop         # 停止守护进程
```

守护进程以独立子进程方式运行，不占用终端。首次调用其他命令时也会自动启动。

### exec — 执行命令

```powershell
pty-agent exec myid -c "python -i -u" -t ">>>" --timeout 10
```

| 参数 | 说明 |
|------|------|
| `-c, --command` | **必填**。要执行的命令字符串（默认经 shell 执行，支持 `&&`、`2>&1`、`\|` 等 shell 语法） |
| `--pty` | 启用完整伪终端（自动拆分命令为列表，走 ConPTY/ConDrv，不支持 shell 语法 `\| && >` 等） |
| `--force-pty-mode` | 强制模式：忽略 `--pty` 的 shell 操作符检测，原样执行 |
| `--shell` | 指定命令解释器：`powershell`（默认，不可用时回退 `cmd`）、`cmd`、`pwsh`、`bash`。仅字符串命令有效，**与 `--pty` 互斥** |
| `-t, --trigger` | 触发条件（正则），匹配后返回输出 |
| `--timeout` | 等待超时秒数（默认 120，可通过 `--default timeout` 临时修改） |
| `--idle-timeout` | 输出静默超时（秒）。程序持续 N 秒无新输出时触发返回 |
| `--idle-after-first-output` | 仅在首次输出后才开始检测静默超时（初始不检测） |
| `--encoding` | 终端编码（utf-8 / gbk），默认自动探测或本次调用记忆 |
| `--newline` | 仅在换行后检查触发 |
| `--full` | 返回全部累积输出而非仅新输出 |
| `--keep-ansi` | 保留终端颜色/样式码（默认过滤；清屏/光标等控制序列始终保留） |

引号处理规则（详见 [`引号处理规则.md`](引号处理规则.md)）：

| Shell | 推荐写法 |
|-------|---------|
| **cmd.exe** | `pty-agent exec test -c "python -c \"import time; print('done')\"" --idle-timeout 3` |
| **PowerShell 5.1 / pwsh 7+** | `pty-agent exec test -c 'python -c "import time; print(''done'')"' --idle-timeout 3` |
| **Git Bash** | `pty-agent exec test -c "python -c \"import time; print('done')\"" --idle-timeout 3` |

> 脚本内置了 `CommandLineToArgvW` 引号修复机制，但 PowerShell 下 `\"` 不转义，
> **推荐 PowerShell/pwsh 用户始终用外层单引号 + 内层双引号**，避免嵌套引号问题。

> 进程崩溃自动返回：`exec`/`send` 等待触发期间若检测到子进程崩溃，**立即返回**，reason 为 `"crashed"`，输出中附带崩溃事件。
>
> 输出静默超时：`--idle-timeout` 检测程序持续 N 秒无新输出时，返回 reason 为 `"idle_timeout"`。

### send — 发送输入

```powershell
pty-agent send myid "print('hello')" -t ">>>"
```

参数同 exec（不含 `--command`），但增加 `input` 位置参数。

**默认 raw 模式**：输入文本原样发送，不做任何转义处理。Windows 路径中的反斜杠（如 `C:\Users\rikka\new_folder`）不会被误转换。

**JSON 转义模式**（`--json-escaping`）：启用后支持标准 JSON 转义序列，适用于需要发送多行代码等场景：

| 转义 | 含义 | 示例 |
|------|------|------|
| `\n` | 换行 | `"for i in range(3):\n    print(i)"` → 两行代码 |
| `\t` | 制表符 | `"print\t1"` |
| `\\` | 反斜杠 | `"C:\\\\path"` |
| `\"` | 双引号 | `"He said \\"hi\\""` |
| `\uXXXX` | Unicode | `"\\u4f60\\u597d"` → `"你好"` |

```powershell
# raw 模式（默认）：路径原样发送
pty-agent send py "open('C:\Users\rikka\new_folder\test.txt')" -t ">>>"

# JSON 转义模式：\n 转为真正换行，发送多行代码
pty-agent send py "for i in range(3):\n    print(i)" --json-escaping -t ">>>"
# PTY 收到:
# for i in range(3):
#     print(i)
# （末尾自动追加换行）
```

### read — 读取输出

```powershell
pty-agent read myid                    # 读取全部新输出
pty-agent read myid --full             # 读取全部累积输出
pty-agent read myid --offset 128       # 从指定字节偏移开始读
pty-agent read myid --lines 5          # 读取最后 5 行
pty-agent read myid --lines 10:20      # 读取第 10~20 行
pty-agent read myid --grep "Error"     # 只显示匹配行
```

| 参数 | 说明 |
|------|------|
| `--full` | 返回全部累积输出而非仅新输出 |
| `--offset <bytes>` | 从指定字节偏移开始增量读取 |
| `--lines <N>` | 最后 N 行；`start:end` 为范围行 |
| `--grep <regex>` | 正则匹配过滤行 |
| `--keep-ansi` | 保留终端颜色/样式码（默认过滤） |
| `--encoding` | 终端编码，默认自动探测或本次调用记忆 |

### list — 列出会话

```powershell
pty-agent list
```

JSON 输出示例：
```json
{"type":"ok","sessions":[{"id":"web","command":"python -i -u","running":true,"pending_events":3}]}
```

自然语言输出示例：
```
ID       命令                       状态    事件
---------------------------------------------------
web       python -i -u              运行中  !3
gcc       g++ main.cpp              已结束
```

`事件` 列显示 `!N` 表示该会话有 N 个待处理事件。

### events — 查看事件

```powershell
pty-agent events myid                          # 所有事件（历史 + 待处理，不消费）
pty-agent events myid --since "18:00"           # 18:00 之后的事件
pty-agent events myid --last 5                  # 最近 5 条事件
pty-agent events myid --since "2026-06-07T18:00:00" --until "2026-06-07T18:30:00"
```

> `--since` / `--until` 支持完整 ISO 8601（如 `"2026-06-07T18:00:00"`）或简写 `"HH:MM"`（自动取当天日期）。

JSON 输出示例：
```json
{"type":"ok","session_id":"myid","pending_events":[
  {"time":"2026-06-22T14:32:15.47","type":"process_spawn","pid":11888,"info":"PID 11888 created","still_active":false},
  {"time":"2026-06-22T14:32:16.89","type":"process_crash","pid":14708,"info":"PID 14708 crashed!","still_active":false}
],"count":2}
```

自然语言输出示例：
```
# ── 事件 (3 项) ────────
# [+]  [22:52:00] PID 11888 创建
# [+]  [22:52:00] PID 14708 创建
# [-]  [22:52:01] PID 14708 退出
# [!!] [22:52:01] 进程崩溃!
#    PID 14708 崩溃! 退出码=1
```

| 标记 | 含义 |
|------|------|
| `[+]` | 进程创建 |
| `[-]` | 进程退出 |
| `[!!]` | 进程崩溃（退出码 ≠ 0） |
| `[W]` | GUI 窗口出现 |

> **存在性检测**：每个事件附带 `still_active` 字段，标明进程/窗口当前是否仍然存活。`process_exit` / `process_crash` 事件固定为 `false`，`process_spawn` 实时检测 PID 是否在进程列表中。

### kill — 终止会话

```powershell
pty-agent kill myid      # 终止会话（KILL_ON_JOB_CLOSE 清理进程树）
```

### closewin — 关闭 GUI 窗口

```powershell
pty-agent closewin myid 0x001403AA       # 十六进制窗口句柄
pty-agent closewin myid 1313450         # 十进制窗口句柄
```

`<hwnd>` 参数支持十进制或 `0x` 十六进制格式。通过 `SendMessage(WM_CLOSE)` 关闭窗口。如果无效，`kill` 会话也可清理。

## 输出格式

### JSON 模式（默认）

所有命令默认输出单行 JSON 到 stdout，便于 AI agent 解析。**JSON 模式下 stdout 仅输出 JSON，不含任何非 JSON 内容**。可通过 `--output-by-natural-language` 切换到自然语言模式。

除命令响应外，以下场景也以 JSON 格式输出到 stdout：

| 场景 | JSON 格式 | 示例 |
|------|-----------|------|
| 守护进程启停信息 | `{"type": "info", "message": "..."}` | `{"type":"info","message":"[pty-agent] 守护进程已启动 (端口 12345)"}` |
| 配置查询（`--show-config`） | `{"type": "config", "content": "..."}` | `{"type":"config","content":"当前调用配置:\n  timeout = 120.0"}` |
| 帮助文本（无子命令） | `{"type": "help", "content": "..."}` | `{"type":"help","content":"usage: pty-agent ..."}` |
| 警告信息 | `{"type": "warning", "message": "..."}` | `{"type":"warning","message":"--idle-after-first-output 需要配合 --idle-timeout 使用"}` |

> 注：守护进程日志（`_logger.warning` 等）不会泄漏到 stderr。当 `DAEMON_LOG_LEVEL=None` 时，所有守护进程 logger 配置 `NullHandler` + `propagate=False`，阻止 Python lastResort handler 输出。

### 自然语言模式

程序实际输出 → **stdout**（纯文本）

元数据/调试信息 → **stdout**（默认纯文本）或 **stderr**（`--color` 启用时终端着色输出）

```
程序输出文本                     ← stdout（实际输出）

# ── session ────────────────────────
# matched | running                ← stderr（元数据）
# session id: 3e8a1f2c
# pty type: conpty
# program: python -i -u            ← 程序名
# started at: 2026-06-08 09:15:30  ← 启动时间
# current time: 2026-06-08 09:18:00← 当前时间
# ── debug ────────────────────────
# process tree: PID 1234 (C:\Python311\python.exe), PID 5678 (C:\Windows\System32\conhost.exe)
# ── 事件 (2 项) ────────
# [+]  [22:52:00] PID 5678 创建
# [-]  [22:52:01] PID 5678 退出
# ── 偏移 ────────────────────
# 172
```

## 进程崩溃自动返回机制

当 `exec` 或 `send` 等待触发时，后台机制实时检测进程状态：

1. **IOCP 实时通知**（Windows Job Object）：进程崩溃时即时推送通知，进程监控器立即设置崩溃标志
2. **监控线程每 2 秒轮询补全**：补充检测 IOCP 未覆盖的进程变更和 GUI 窗口
3. `wait_for_trigger` 轮询循环（每 0.1s）优先检测崩溃标志，检测到后立即返回 `reason="crashed"`
4. 响应中 `debug.pending_events` 包含 crash 事件详情

**不依赖读线程 EOF 检测**，崩溃后即时返回，无需等待超时或输出触发。

## FAQ

| 问题 | 解决 |
|------|------|
| `python -u` 不出 `>>>` | 用 `python -i -u`（强制交互模式） |
| 会话卡住 | `pty-agent kill <id>` |
| 守护进程无响应 | `pty-agent stop; pty-agent start` |
| 中文乱码 | `--encoding gbk`（本次调用记忆） |
| 端口冲突 | 端口是动态分配的，用 `pty-agent stop; pty-agent start` 即可更换端口 |
| send 触发总超时（reader 竞态） | v0.6.0 已修复：内部已将 `set_trigger` 移至 `write_input` 之前，避免 reader 抢读响应后触发置在缓冲区末尾 |
| 进程树显示 PID 3/PID 0 等诡异值 | v0.6.0 已修复：`JOBOBJECT_BASIC_PROCESS_ID_LIST` 解析错误（错用扁平数组 + `c_size_t` 类型），改用正确结构体和 `DWORD` 数组 |
| 需要"程序执行完静默后返回" | 使用 `--idle-timeout <秒>`，程序 N 秒无输出即返回。无需配合 `--trigger`。配合 `--idle-after-first-output` 可先等程序启动再检测静默 |
| 需要完整伪终端（ConPTY） | 使用 `--pty`，命令自动拆为列表传给后端，响应 `program.pty_type` 为 `win-conpty`（ConDrv 直连因 I/O 不完整已临时禁用）。含 shell 操作符时加 `--force-pty-mode` 跳过检测 |
| 响应中的 `pty_type` 是什么 | 标识当前会话使用的 PTY 后端：`subprocess`（管道回退）、`win-conpty`（CreatePseudoConsole）、`unix-pty`（Unix PTY）。ConDrv 直连因 I/O 不完整已禁用 |
| 进程树显示完整路径 | `debug.processes` 中 `path` 字段为可执行文件的完整路径（如 `C:\Python311\python.exe`），自然语言模式显示为 `PID 1234 (C:\Python311\python.exe)` |
| send/exec 返回值带哪些信息 | 返回 v4 规范：`trigger_matched`（bool）+ `reason`（字符串）+ `program`（含 command/start_time/pty_type）+ `debug.pending_events`（待处理事件） |
| PowerShell 下 `exec -c "python -c \"...\""` 报错 | PowerShell 的 `\"` 不会转义引号（`\` 是字面量），导致参数被拆分。推荐外层用单引号 `'...'` 或双写 `""`。详见 [`引号处理规则.md`](引号处理规则.md) |
| `send` 输入中的 `\n` 不换行 | 默认 raw 模式不处理转义，需加 `--json-escaping` 启用 JSON 转义解码 |
| 响应中 debug 信息过多 | 使用 `--no-debug` 或 `--default debug off` 禁用进程树/GUI 窗口/事件输出 |
